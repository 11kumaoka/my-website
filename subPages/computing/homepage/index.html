Git Hub page

作り方

Git Hub pageはGitHubのリポジトリを使ってWebページをホストするサービスです。 GitHubのリポジトリを作成し、そこにHTMLファイルを配置することで、簡単に無料でWebページを公開できます。 作成方法はGitHubのページで書いてあるので、そちらを参照してください。 ひょっとすると、このページの方が参考になるかもです。 GitHub pageは静的なWebサイトをホストするのに適しており、個人のポートフォリオやプロジェクトのドキュメントなどに利用されます。 GitHub pageを利用するには、リポジトリの設定でGitHub Pagesを有効にし、公開するブランチを指定します。 その後、リポジトリのURLにアクセスすることで、公開されたWebページを閲覧できます。

インデクスへの登録方法

GitHub pageのURLは、通常、https://username.github.io/repository-name/の形式になります。 初めはGoogleのような検索エンジンに登録されていないため、公開後は自分でURLを共有する必要があります。 Google Search Consoleを使って、サイトマップを送信することもできます。 "https://username.github.io/my-pageをURL プレフィックスにいれて、 html形式のファイルが生成されるのでそれをダウンロードし、自身のウェブサイトのトップに配置したのち、 GitHubのリポジトリにアップロードすることで、サイトマップを登録できます。 その後、Search Consoleに戻って「確認」をクリックします。 しばらくなり、数日するとGoogleの検索結果に反映されるようになります。

ページのざっくりな構造

基本的には大きなタグで囲んでいき、箱の中に箱を作りページを作っていきます。 一番大きな箱は<html>タグです。 その中に<head>タグや<body>タグを入れていきます。 <head>には設定やメタ情報、CSSのリンクなどを入れます。 <body>には実際のコンテンツを入れます。 その中にさらに<header>や<main>、<footer>などのセクションを作り、内容を整理していきます。 例えば、<header>にはページのタイトルやナビゲーションを入れ、<main>にはメインコンテンツを入れます。 <footer>には著作権情報やリンクを入れることが一般的です。 また、<section>や<div>タグを使って、セクションごとに内容を分けることもできます。

箱のサイズや空間について

文字の基礎

フォントサイズ

<span style="font-size: 16px;">16px</span> は標準的なフォントサイズです。
<span style="font-size: 1em;">1em</span> は親要素のフォントサイズに対する相対単位です。

クラス

既存のクラスの改造や自身のクラスを使うことで、スタイルを簡単に適用できます。 例えば、<span class="ownStyle">ownStyle</span> というクラスを定義して、特定のスタイルを適用できます。

リンク

リンクは <a href="https://www.example.com">ここをクリック</a> で開くことができます。
<a href="https://www.example.com" target="_blank">新しいタブで開くリンク</a> も可能です。
<a href="https://www.example.com" target="_blank" rel="noopener noreferrer">セキュリティを考慮したリンク</a> も書けます。

内部ページには単にそのパスを書けばOKです：
<a href="/main/subpage/">下層ページへのリンクへ飛ぶ</a>

また、ページ内の特定の場所に飛ぶこともできます。まず、飛びたい場所にIDを設定します（例：id="exam1"）。
<section class="hoge" id="exam1">ほげほげ</section>
その場所に対してリンクを設定します：
<a href="#exam1">ほげほげに飛ぶリンク</a>
別のページの特定の場所に飛ぶこともできます：
<a href="/main/subpage/#exam1">別のページのほげほげに飛ぶリンク</a>
